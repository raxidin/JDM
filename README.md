# Java Download Manager (JDM)

基于 Java 8 的多线程下载管理器，类似 IDM（Internet Download Manager），支持 HTTP/HTTPS 多线程分片下载，提供 Swing GUI 界面。

## 功能特性

- **多线程分片下载** — 文件自动分片，4~16 线程并发下载，充分利用带宽
- **智能降级** — 服务器不支持 Range 时自动切换为单线程下载
- **断点续传** — 暂停后可恢复下载（同一会话内）
- **实时进度** — 进度条、下载速度、文件大小实时显示
- **重试机制** — 网络波动自动重试（最多 3 次，递增等待）
- **重定向跟随** — 自动处理 HTTP 301/302/303/307/308 重定向
- **剪贴板粘贴** — 支持 Ctrl+V 一键粘贴 URL

## 项目结构

```

├── downloader/
```

### 关键设计决策

| 设计点 | 决策 | 原因 |
|--------|------|------|
| Range 支持检测 | 只信任探针返回的实际 `206` 状态码 | 部分服务器 `Accept-Ranges: bytes` 头存在但不实际支持，会导致数据覆盖 |
| 探针方式 | `GET Range: bytes=0-0` | HEAD 请求部分 CDN 返回 502，GET+Range 通用且不消耗流量 |
| 进度追踪 | 每片独立 `AtomicLong`，重试归零 | 避免重试时进度重复累加 |
| 重试策略 | 3 次，递增等待 (1s → 2s) | 覆盖瞬时网络抖动，不无限重试 |
| 超时设置 | 连接 15s / 读取 60s | 大文件下载中 CDN 间歇断流 |

## 运行

### 环境要求

- JDK 8+
- 无外部依赖（仅使用 JDK 标准库）

### 编译运行

```bash
# 编译
javac -d out/production/installler src/Main.java src/downloader/*.java src/ui/*.java

# 运行
java -cp out/production/installler Main
```

或在 IntelliJ IDEA 中直接运行 `Main.java`。

### 使用方式

1. 输入或粘贴下载 URL
2. 选择线程数（默认 4，建议小文件 1~4，大文件 4~8）
3. 选择保存目录
4. 点击 **+ Add Download** 开始下载
5. 右键或右侧按钮可暂停/恢复/取消/移除任务
